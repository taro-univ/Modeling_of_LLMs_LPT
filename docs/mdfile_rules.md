## GitHub 数式記述・レンダリングルール

GitHub上で数式をエラーなく、綺麗に表示させるための標準記法ルールです。

### 1. 基本の記法
GitHubのMarkdownエンジンでMathJaxを有効にするため、以下の2つの形式を使い分けてください。

- **インライン表示（文中に埋め込む場合）**
  `$E=mc^2$` のように、1つの `$` で数式を囲みます。
  ※記号の直後にスペースを入れないでください（例: `$ E=mc^2 $` は失敗する可能性があります）。

- **ブロック表示（独立した行で表示する場合）**
  `$$` で囲みます。**数式の上下に必ず1行の空行を入れる**ことで、レンダリングの失敗を防げます。
  ```markdown
  （空行）
  $$
  H = -J \sum_{\langle i,j \rangle} \cos(\theta_i - \theta_j)
  $$
  （空行）

  ### 2. エラー回避（アンダースコア `_` の扱い）
Markdownの仕様（`_text_` で斜体になる）との干渉を避けるため、以下の対策を徹底してください。

- **エスケープの利用**
  数式内でアンダースコアを多用する場合は、`\_` と記述するか、数式全体をブロック表示（`$$`）にしてください。
- **\text{} の活用**
  変数名やラベルにアンダースコアが含まれる場合は、`\text{dtype\_bytes}` のように `\text{}` で囲みます。これにより、Markdownエンジンによる誤認を防ぎ、数式フォントではなく標準フォントで正しく表示されます。
